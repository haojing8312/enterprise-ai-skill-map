入选理由：
- 痛点靠近现金流：首封回复、报价、样品、补资料之后，热线索容易在邮箱里冷掉。
- 与 AI 询盘分拣员不重复：本岗位处理已进入销售流程后的下一步推进、提醒和草稿。
- 低成本试点：指定邮箱标签、50–100 条脱敏历史邮件、一张 CRM/表格队列即可启动。
- 工作机理清楚：邮箱/CRM → 线程摘要 → 跟进判断 → 草稿生成 → CRM/日报 → 人工发送。

## 公开资料与工具栈验证

- n8n + Gmail/Outlook 节点：邮件触发、摘要、草稿、表格/CRM 写入；第一周不自动发送。
- Baserow / NocoDB：承接跟进队列、草稿审核表、规则表和验收表；客户资料需权限控制。
- HubSpot / Pipedrive：承接联系人、公司、交易、任务、备注；AI 初期只写备注和任务。
- LangChain / Open Deep Research：公开客户背景核验、来源引用、结构化输出；公开研究不能替代尽调。
- Gmail API / Microsoft Graph Mail API：工程化读取线程、创建草稿、同步邮件活动；权限最小化。
- 合规参考：FTC CAN-SPAM Guide、Google Bulk Sender Guidelines。
