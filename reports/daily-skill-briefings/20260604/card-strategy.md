# 2026-06-04｜Skill 早报卡片策略

## 今日主题

营销、SEO/GEO、内容分发、ASO：给增长团队的一组可试 Skill。

今天不讲“AI 又多聪明”，只讲一个判断：这些 repo 已经从单个提示词，长成了可安装、可审计、可复用的小工具箱。读者看完卡片，应该知道自己要试哪一个，也知道哪里不能乱用。

## 主判断

今天的 5 个入选项，适合做成“增长工具箱”主题：

- `marketingskills` 做总入口：营销 Skill 大库，星标和覆盖面最强。
- `claude-seo` 做可靠审计：SEO/GEO 证据、结构、可复现性最好。
- `geo-seo-claude` 做新趋势：AI 搜索引用、llms.txt、citability，话题清楚。
- `GEOFlow` 做系统场景：中文/多语言内容资产、多站分发、人工审核。
- `aso-skills` 做垂直补位：App Store 增长，和 SEO/GEO 不撞车。

被挡掉的 `OpenOutreach` 不放进卡片，只在底部判断里留一句安全提醒：外联自动化有平台规则和合规风险，别拿个人账号硬试。

## 读者画像

- 独立开发者：想把营销、SEO、ASO 工作拆给 agent 做初稿和检查。
- 出海小团队：缺专职 SEO/GEO 或增长运营，需要一套能照着试的清单。
- 内容负责人：关心 AI 搜索、知识库内容、多站发布，怕自动发布失控。
- App 团队：需要关键词、竞品、商店页优化建议，但仍要人工把关。

## 卡片顺序

1. **营销大库**：先给最大、最容易理解的入口，降低阅读门槛。
2. **SEO 体检**：把网站审计需求接上，强调证据和优先级。
3. **AI 搜索**：点出 GEO 的新变化，用“被 AI 引用”讲清楚。
4. **内容流水线**：把 Skill 从单点工具扩展到内容生产系统。
5. **应用商店**：收在 ASO 场景，给移动应用团队一个明确入口。

## 上图文字策略

- 标题只讲人话：**今天 5 个增长 Skill**。
- 每张卡片 1 个短标题 + 1 句短说明，单句尽量 12-18 个字。
- 不堆 repo 全名，repo 名放在卡片小字或配套正文中。
- 卡片里不写星标数，避免视觉拥挤；证据留给正文或备注。
- 底部只放一个判断：能试，但不能自动发布、不能违规外联。

## 视觉策略

- 版式：竖版移动端早报，顶部标题区 + 5 张圆角卡片 + 底部判断。
- 字号：标题大，卡片短句不小于 14pt；每张卡片不超过两行主文字。
- 气质：干净、科技感、轻量，不做复杂信息表。
- 色彩：深色背景，5 张卡片用不同低饱和色块区分。
- 图标建议：营销漏斗、放大镜、AI 引用符号、内容管道、应用商店。

## 安全边界

- SEO/GEO/ASO 建议只能当诊断和初稿，不能写成排名、下载量、收入承诺。
- 抓取网站或商店数据时，只处理有权访问的公开内容，遵守 robots、平台条款和版权要求。
- API key、商店账号、WordPress 密钥由用户自管，不写入公开 repo。
- 多站发布要保留人工审核、发布节奏和回滚办法。
- 冷邮件、LinkedIn 外联、批量私信类能力不在今日推荐范围内。
