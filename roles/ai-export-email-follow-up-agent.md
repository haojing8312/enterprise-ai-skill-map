# AI 外贸邮件跟进员岗位地图：让客户跟进不再靠业务员记性

> 岗位英文名：AI Export Email Follow-up Agent  
> slug：`ai-export-email-follow-up-agent`  
> 适用对象：跨境 B2B 外贸公司老板、销售主管、制造出口企业负责人  
> 核心边界：生成跟进队列、邮件摘要、草稿、CRM 备注和日报；不自动报价、不承诺交期、不签合同、不绕过退订和平台规则。

---

## 一、岗位描述：这个岗位到底做什么？

**AI 外贸邮件跟进员**不是一个“群发冷邮件机器人”。更准确地说，它是外贸销售团队的“跟进副驾驶”：

- 每天从邮箱和 CRM 里找出今天该跟进的客户；
- 把长邮件线程总结成业务员一眼能看懂的客户上下文；
- 判断客户处于询盘、补资料、报价后、样品后、沉默唤醒还是待关闭阶段；
- 生成英文或多语言跟进邮件草稿；
- 标出需要人工确认的价格、交期、付款、合同、合规风险；
- 把摘要、任务、下次跟进日期写回 CRM 或销售表；
- 给老板和销售主管输出每日跟进日报。

一句话：**它解决的是“客户没人及时跟、业务员忘记跟、主管看不见谁没跟、邮件草稿质量不稳定”的问题。**

---

## 二、老板痛点：为什么外贸老板会需要这个岗位？

外贸销售的损失，很多不是输在价格，而是输在跟进动作断掉：

1. **热询盘没有及时跟进**：客户问了规格、样品、MOQ，业务员当天忙，过两天再回，客户已经找了别人。
2. **报价后没人追踪**：报价发出去了，但没有提醒业务员 3 天后问客户是否需要调整规格、样品或物流方案。
3. **邮件线程太长**：客户来回沟通十几封，新人或主管接手时不知道上次说到哪。
4. **CRM 变成摆设**：业务员不愿意填，主管看不到下次跟进日期、客户阶段和未回复原因。
5. **英文跟进邮件质量不稳定**：有人写得太硬，有人写得太空，有人只会一句 “Any update?”。
6. **客户被重复打扰**：同一个客户被不同业务员跟，或已经拒绝/退订还继续发。
7. **老板只看到结果，看不到过程**：月底才发现很多客户从来没有被二次跟进。

AI 外贸邮件跟进员的价值，就是把这些“靠人记、靠人翻、靠人复制”的动作变成可见、可追踪、可验收的日常流程。

## 三、适合哪些公司？

### 最适合

- 每天有 20–300 封销售相关邮件的跨境 B2B 公司；
- 有多个业务员，客户跟进质量参差不齐；
- 产品有一定复杂度，客户常问规格、认证、包装、样品、MOQ、交期、目标港；
- 已经使用 HubSpot、Pipedrive、Zoho、Salesforce、企业微信表格、Google Sheet、Baserow 或 NocoDB；
- 老板或销售主管愿意定义客户等级、跟进节奏、禁发规则和报价审批规则；
- 想先做低风险试点：只读邮箱、生成草稿、人工发送。

### 暂时不适合

- 邮件量很小，老板亲自盯几个大客户；
- 没有 CRM，也没有统一客户表；
- 公司希望 AI 直接谈价格、承诺交期、签合同；
- 每封邮件都包含高度敏感合同、图纸、未公开报价；
- 团队不愿意验收 AI 草稿，只想“一键全自动成交”。

## 四、它每天接收什么？输出什么？

### 每天接收的输入

| 输入 | 例子 | 注意事项 |
|---|---|---|
| 邮箱线程 | 客户询价、补资料、报价后追问、样品反馈 | 只接入销售邮箱或指定标签，不接入私人邮箱 |
| CRM 数据 | 客户等级、交易阶段、上次活动、下次跟进日期 | 初期只读关键字段 |
| 询盘分拣结果 | 产品、国家、数量、客户等级、负责人 | 可与“AI 询盘分拣员”衔接 |
| 产品公开资料 | 官网产品页、公开目录、FAQ、认证说明 | 不使用内部成本、底价、未公开 SKU |
| 跟进规则 | A/B/C 客户节奏、禁发客户、敏感国家 | 由主管维护，AI 只执行 |
| 邮件模板与语气 | 公司签名、常见 CTA、英文风格 | 模板要人工审核 |
| 公开客户资料 | 客户官网、公开公司页面 | 只做背景辅助，不猜测采购能力 |

### 每天输出的结果

| 输出 | 给谁 | 用来做什么 |
|---|---|---|
| 今日跟进清单 | 业务员/主管 | 知道今天必须跟谁、为什么跟 |
| 邮件线程摘要 | 业务员 | 3 分钟看完客户上下文 |
| 跟进邮件草稿 | 业务员 | 修改后发送，不从零写 |
| 缺失信息清单 | 业务员/技术 | 知道下一封邮件要问什么 |
| CRM 备注和任务 | 销售主管 | 让跟进动作进入系统 |
| 风险提醒 | 主管 | 退订、重复打扰、敏感国家、可疑附件、价格承诺风险 |
| 每日跟进日报 | 老板 | 看团队是否漏跟、慢跟、乱跟 |

## 五、可接手任务：它能接手的 8 个任务

### 1. 今日应跟进清单

AI 每天早上检查 CRM 和邮箱，列出：

- 今天到期跟进的客户；
- A/B 级客户超过 24/48 小时未处理；
- 报价后 3–5 天未回复；
- 样品寄出后未确认；
- 展会后未二次触达；
- 老客户超过固定周期未沟通。

输出不是一句“记得跟进”，而是：客户是谁、为什么今天要跟、上次说了什么、建议怎么跟。

### 2. 邮件线程摘要

很多外贸客户一聊就是十几封邮件。AI 可以把线程整理成：

- 客户公司和联系人；
- 询问的产品、规格、数量、用途；
- 客户已经提供的信息；
- 我方已经承诺过的内容；
- 当前卡点；
- 下一步建议。

主管接手或新人接手时，不用从头翻邮件。

### 3. 跟进邮件草稿

根据不同场景生成不同草稿：

- 首次询盘后的补资料邮件；
- 报价后跟进；
- 样品寄出后确认；
- 展会见面后跟进；
- 14 天未回复唤醒；
- 老客户复购提醒；
- 节假日前交期确认。

重点：**草稿默认不自动发送，必须业务员确认。**

### 4. 缺失信息和补问清单

AI 可以自动发现客户还没给的信息：

- 具体型号/规格；
- 采购数量；
- 包装要求；
- 目标港；
- 认证要求；
- 使用场景；
- 采购时间；
- 是否需要样品。

然后把这些问题写成礼貌、简短的英文邮件。

### 5. 公开客户背景卡片

对于陌生客户，AI 可以基于公开资料生成背景卡：

- 公司是否有公开官网；
- 主营业务是否与我方产品相关；
- 邮箱域名是否匹配公司；
- 可能是批发商、零售商、工程商、制造商还是代理；
- 哪一句可以用于个性化开场。

注意：如果没有公开证据，就写“未找到”，不能编造客户规模和采购能力。

### 6. CRM 备注与任务写回

AI 可以把每封关键邮件整理成 CRM 备注：

- 本轮客户问了什么；
- 我方回复了什么；
- 下一步动作；
- 下次跟进日期；
- 是否需要主管/技术/老板确认。

这样老板看到的不只是“跟进中”，而是能知道到底卡在哪里。

### 7. 沉默客户分层唤醒

不是所有未回复客户都应该一样催。AI 可以分层：

- 7 天未回：轻提醒，确认是否收到资料；
- 14 天未回：补充价值点或问是否需要调整规格；
- 30 天未回：给替代方案或询问项目是否暂停；
- 60 天未回：低频培育或关闭；
- 明确拒绝/退订：停止跟进。

### 8. 每日跟进日报

老板每天可以看到：

- 今天应跟进客户数；
- 已生成草稿数；
- 已人工发送数；
- 客户回复数；
- 超时未处理客户；
- A 级客户当前状态；
- 需要老板/主管决策的客户。

这会把销售跟进从“黑箱”变成“可管理流程”。

## 六、能力模块：这个 AI 员工需要哪些能力？

### 模块 1：客户上下文理解

能读懂邮件线程和 CRM 记录，提取客户需求、历史承诺、当前卡点。最重要的是：不编造没有出现的信息。

### 模块 2：跟进节奏和优先级判断

根据客户等级、交易阶段、未回复时长、产品匹配度和上次动作，判断今天是否应该跟进。

### 模块 3：外贸邮件草稿生成

能写短、清楚、有 CTA 的英文/多语言邮件。不是套模板，而是结合客户上下文写。

### 模块 4：CRM 和表格写回

能把 AI 产出的摘要、任务、日期、负责人、风险标记写回系统，让主管能验收。

### 模块 5：合规和风险控制

能识别退订、拒绝、投诉、敏感国家、可疑附件、过度跟进、承诺性语言，并触发人工审核。

## 七、工作机理：输入 → 处理 → 输出 → 人工验收

```text
输入
  ├─ 销售邮箱/共享邮箱
  ├─ CRM 联系人、公司、交易、任务
  ├─ 产品公开资料和 FAQ
  ├─ 跟进节奏与禁发规则
  └─ 公司邮件模板和签名

处理
  ├─ 过滤掉退订、投诉、内部邮件、垃圾邮件
  ├─ 读取邮件线程并做摘要
  ├─ 判断客户阶段和跟进优先级
  ├─ 识别缺失信息和风险项
  ├─ 生成邮件草稿
  └─ 写入 CRM/表格/日报

输出
  ├─ 今日跟进队列
  ├─ 邮件摘要
  ├─ 英文/多语言草稿
  ├─ 缺失信息清单
  ├─ CRM 备注和任务
  └─ 老板日报

人工验收
  ├─ 业务员确认事实、修改语气、点击发送
  ├─ 主管抽查优先级和 CRM 写回
  ├─ 老板看漏跟、慢跟、回复趋势
  └─ 合规/负责人处理敏感问题
```

## 九、主管验收指标：主管怎么验收？

建议第一周不要只看“生成了多少封邮件”，而要看质量和风险。

| 验收指标 | 第一周建议目标 |
|---|---:|
| 邮件线程摘要准确率 | ≥ 85% |
| 产品、数量、国家、联系人等字段准确率 | ≥ 85% |
| 今日跟进队列不漏 A/B 级到期客户 | ≥ 90% |
| 邮件草稿可用率 | ≥ 60% |
| 业务员从零写邮件时间下降 | ≥ 30% |
| CRM 备注和任务写回准确率 | ≥ 90% |
| 未授权自动发送 | 0 次 |
| 退订/拒绝客户继续跟进 | 0 次 |
| 价格、交期、合同误承诺 | 0 次 |

如果一周后草稿可用率不高，但摘要准确、跟进队列准确，也值得保留。因为“提醒该跟谁”和“总结上次说了什么”本身已经能减少漏跟。

## 十、不能自动化事项：哪些事情不能交给它自动做？

1. **不能自动报价**：价格、折扣、MOQ、账期、运费、税费必须人工确认。
2. **不能承诺交期和库存**：生产排期、样品时间、库存锁定要人工确认。
3. **不能确认订单和合同**：PI、PO、合同、变更条款必须人工审核。
4. **不能独立做出口管制或制裁判断**：AI 只能提示风险。
5. **不能自动群发冷邮件**：尤其不能没有退订、没有频率控制、标题误导。
6. **不能忽略客户退订和拒绝**：明确拒绝后应停止跟进。
7. **不能读取私人邮件和敏感附件**：内部成本、底价、图纸、合同不要进入通用 AI 流程。
8. **不能自动打开可疑附件**：压缩包、宏文件、可执行文件要隔离。
9. **不能把猜测当事实**：公开资料找不到就写“未找到证据”。
10. **不能替业务员建立虚假关系**：不能写“as discussed”这类实际没发生过的说法。

## 十一、第一周怎么试点？

### 试点原则

- 小范围：只选 1 个邮箱标签、2–3 名业务员、50–100 个历史客户线程。
- 低风险：只生成草稿，不自动发送。
- 可追溯：每条摘要和草稿都能回到原邮件。
- 有验收：业务员每天打分，主管抽查。

### 5 天安排

| 天数 | 做什么 | 当天验收 |
|---|---|---|
| Day 1 | 定义字段、客户等级、跟进节奏、禁发规则、邮件语气 | 主管确认规则表和风险边界 |
| Day 2 | 接入邮箱/CRM，只做线程摘要和表格写入 | 50 条历史邮件可追溯，摘要准确率初测 ≥ 80% |
| Day 3 | 生成今日跟进队列和缺失信息清单 | 不漏掉 A/B 级到期客户 |
| Day 4 | 生成英文跟进草稿和人工确认项 | 草稿可用率 ≥ 50%，无价格/交期误承诺 |
| Day 5 | 用真实新增邮件跑完整流程，输出老板日报 | 形成继续扩大、调整或暂停的决策 |

### 一周后怎么决策？

- **可以扩大**：摘要准确、草稿可用、无合规事故、业务员愿意用。
- **只保留部分功能**：草稿一般，但跟进提醒和摘要好用。
- **需要暂停**：出现误发、乱写 CRM、价格承诺、退订违规。

## 十二、可用工具栈：从轻到重

### 1. n8n + Gmail/Outlook

- 链接：n8n 文档 https://docs.n8n.io/；Gmail 节点 https://docs.n8n.io/integrations/builtin/app-nodes/n8n-nodes-base.gmail/；Outlook 节点 https://docs.n8n.io/integrations/builtin/app-nodes/n8n-nodes-base.microsoftoutlook/
- 用途：搭建邮件触发、AI 摘要、草稿生成、表格/CRM 写入、提醒流程。
- 适合谁：想快速试点、公司有运营或 IT 同事的外贸团队。
- 风险边界：第一周只读和生成草稿，不自动发送。

### 2. Baserow / NocoDB

- 链接：Baserow https://baserow.io/docs；NocoDB https://github.com/nocodb/nocodb
- 用途：做跟进队列、草稿审核表、规则表、验收表。
- 适合谁：CRM 不完善，想先跑 MVP 的中小团队。
- 风险边界：控制客户资料权限，不放合同、成本、底价。

### 3. HubSpot / Pipedrive

- 链接：HubSpot CRM https://www.hubspot.com/products/crm；Pipedrive API https://developers.pipedrive.com/docs/api/v1
- 用途：承接联系人、公司、交易、任务、备注，让主管看到跟进过程。
- 适合谁：已有 CRM 或准备规范销售流程的团队。
- 风险边界：AI 初期只写备注和任务，不自动改金额、阶段、成交概率。

### 4. LangChain / Open Deep Research

- 链接：LangChain https://docs.langchain.com/；Open Deep Research https://github.com/langchain-ai/open_deep_research
- 用途：做客户公开背景核验、来源引用、结构化输出和人工审核流程。
- 适合谁：有工程能力、需要把公开研究接入销售流程的团队。
- 风险边界：公开研究不能替代尽调；没有证据就不能写成事实。

### 5. Gmail API / Microsoft Graph Mail API + 销售邮件模板参考

- 链接：Gmail API https://developers.google.com/workspace/gmail/api/guides；Microsoft Graph Mail API https://learn.microsoft.com/en-us/graph/api/resources/mail-api-overview；Pipedrive 邮件模板参考 https://www.pipedrive.com/en/blog/cold-email-templates
- 用途：工程化读取线程、创建草稿、同步标签和邮件活动；参考销售邮件结构。
- 适合谁：邮箱系统标准化、希望深度集成的团队。
- 风险边界：要遵守发信平台规范和退订规则；创建草稿不等于自动发送。

补充合规参考：FTC CAN-SPAM Guide https://www.ftc.gov/business-guidance/resources/can-spam-act-compliance-guide-business；Google Bulk Sender Guidelines https://support.google.com/a/answer/81126?hl=en

## 十三、老板可以怎样给这个岗位定 KPI？

不要给它定“成交额”。AI 外贸邮件跟进员不是业务员，它应该对流程质量负责：

- A/B 级客户是否按时提醒；
- 邮件摘要是否准确；
- 草稿是否可用；
- CRM 是否及时更新；
- 是否减少业务员写邮件时间；
- 是否减少漏跟；
- 是否保持 0 误发、0 退订违规、0 价格/交期误承诺。

真正的经营目标是：让业务员少做机械活，多做成交判断。

## 十四、建议的试点数据表

老板可以要求团队建一张“AI 跟进队列表”：

| 字段 | 说明 |
|---|---|
| 客户公司 | company |
| 联系人/邮箱 | contact/email |
| 国家 | country |
| 产品 | product_interest |
| 客户等级 | A/B/C/待判断 |
| 邮件摘要 | thread_summary |
| 客户最后回复时间 | last_customer_reply_at |
| 我方最后发送时间 | last_sales_sent_at |
| 建议跟进日期 | followup_due_at |
| 为什么今天跟进 | followup_reason |
| 缺失信息 | missing_info |
| 草稿主题/正文 | draft_subject/draft_body |
| 人工确认项 | price/lead time/payment/compliance |
| 风险标记 | unsubscribe/sensitive country/attachment/over-followup |
| 负责人 | owner |
| 状态 | 待审/已发送/暂缓/停止/关闭 |
| 原邮件/CRM 链接 | source links |

这张表是老板验收 AI 员工的仪表盘。

## 十五、最终定位

AI 外贸邮件跟进员的价值，不是让机器替公司谈判，而是让销售团队的跟进动作不再靠记忆和自觉。

它应该像一个每天准时上班的销售助理：

- 早上列出今天该跟谁；
- 中午提醒谁超时未处理；
- 下午把草稿和 CRM 备注准备好；
- 晚上给老板一张日报；
- 遇到价格、交期、合同、退订、敏感国家，立即停下来请人判断。

如果第一周只做到“少漏跟、少翻邮件、少从零写草稿”，这个岗位就已经值得继续试点。
