# AI 询盘分拣员 / AI Inquiry Triage Agent

> 出海企业的销售线索运营数字员工：把邮箱、官网表单、平台通知里的新询盘，先整理成可判断、可分派、可追踪的线索队列，再交给业务员和销售主管确认。

## 岗位一句话

AI 询盘分拣员每天读取指定询盘入口，把客户需求抽成结构化字段，识别重复和风险，按 A/B/C/Risk 排队，匹配产品线与负责人，并生成销售可编辑的首封回复草稿和老板日报。

## 适合谁

1. **B2B 外贸公司 / 外贸工厂**：每天 20–300 条询盘，回复速度影响成交。
2. **机械、建材、汽配、消费品、包装、原材料出口团队**：产品线多，询盘常带规格、数量、认证和目的港。
3. **有官网表单、询盘邮箱、Alibaba / Made-in-China / Global Sources 通知邮件的团队**。
4. **销售主管想减少漏跟进、重复录入和垃圾询盘占用时间的团队**。

不适合：询盘量很低；每封邮件都涉及高度定制合同、非公开报价、特殊技术承诺；公司没有 CRM、表格或看板承接分拣结果。

## 老板痛点

- 询盘进邮箱后，谁先看、谁该回、谁已经回了，经常说不清。
- A 级线索夹在几十封普通邮件里，业务员忙完才发现已经慢了半天。
- 销售把时间花在复制邮件、查公司、问缺失资料、辨别垃圾询盘上。
- 主管只能靠群里截图管理，老板看不到当天 A/B/C 线索分布。
- 垃圾询盘、重复询盘、可疑附件和敏感地区线索没有统一标记。

## 这个数字员工每天接什么输入

| 输入 | 例子 | 使用边界 |
|---|---|---|
| 指定询盘邮箱 / Gmail 标签 | 邮件正文、主题、发件人、附件文件名 | 只读询盘专用入口，不读老板私人邮箱 |
| 官网表单 | 产品、数量、姓名、公司、电话、留言 | 表单字段标准化，敏感字段脱敏展示 |
| B2B 平台通知邮件 | 平台发来的询盘摘要和链接 | 初期只处理通知摘要，不登录后台批量抓取客户资料 |
| CRM 新线索 | HubSpot / Pipedrive 中的新线索 | 试点期只写备注、标签、任务提醒，不改金额和阶段 |
| 公司公开产品资料 | 官网产品页、公开目录、FAQ、认证说明 | 不使用未发布报价单、内部成本和合同条款 |
| 规则表 | 产品线、负责人、禁售地区、垃圾关键词 | 由销售主管维护，AI 只读取规则 |

## 产出什么结果

- 线索分拣表：公司、联系人、国家、产品、规格、数量、认证、附件风险、原文链接。
- A/B/C/Risk 等级：优先级、原因、风险标记、建议下一步。
- 产品线与负责人分派：匹配区域销售、产品工程师或主管审核池。
- 待补资料清单：缺规格、数量、用途、目标港、认证、采购时间等。
- 首封回复草稿：业务员可编辑，不含报价、交期、库存和付款承诺。
- 老板日报：今日询盘数、A级线索、未分派、风险项、超时未回。

## 可接手任务

1. 新询盘自动建档，把邮件/表单转成统一字段。
2. A/B/C/Risk 线索评级，解释每条评级证据。
3. 近 30/60/90 天重复询盘识别。
4. 按产品线、国家、客户类型分派负责人。
5. 自动列出需要客户补充的问题。
6. 只用公开网页做公司背景核验，并保留来源 URL。
7. 起草保守英文首封回复，交业务员人工发送。
8. 生成销售主管和老板看的每日线索简报。

## 能力模块与相关 Skill 工具栈

| 能力模块 | 推荐工具 / Skill | 适合做什么 | 风险边界 |

## 工作机理

```text
询盘邮箱 / 官网表单 / 平台通知 / CRM 新线索
        ↓
读取指定入口，过滤私人邮箱、合同、报价、敏感附件
        ↓
抽取公司、国家、产品、规格、数量、认证、目的港、附件风险
        ↓
按邮箱、域名、公司名、产品关键词做去重
        ↓
根据产品匹配、信息完整度、采购时间、风险规则评级 A/B/C/Risk
        ↓
匹配产品线、区域和负责人，生成待补资料问题与回复草稿
        ↓
写入分拣表 / CRM 备注 / 内部提醒
        ↓
销售主管抽查，业务员人工回复，老板看日报
```

## 可复制任务提示词

### 1. 询盘结构化提取

```text
你是跨境贸易公司的询盘分拣助理。请只根据给定询盘内容抽取字段，不要猜测，不要补充未出现的信息。

【询盘原文】
{{inquiry_text}}

【发件人信息】
邮箱：{{sender_email}}
姓名：{{sender_name}}
邮件主题：{{subject}}
收到时间：{{received_at}}
附件文件名：{{attachment_names}}

请输出严格 JSON：
{
  "company_name": "",
  "contact_name": "",
  "email": "",
  "country_or_region": "",
  "product_requested": "",
  "specification": "",
  "quantity": "",
  "target_price_or_budget": "",
  "delivery_time": "",
  "destination_port_or_country": "",
  "certification_required": "",
  "intended_use": "",
  "attachments_summary": "",
  "missing_fields": [],
  "language": "",
  "source_quote": "请摘录最能证明客户需求的原文句子"
}

规则：原文没有的信息填空字符串或空数组；不要输出报价、承诺、推测；如果附件文件名像压缩包、宏文件、可执行文件，请标注“需人工安全检查”。
```

### 2. A/B/C/Risk 线索评级

```text
你是销售主管的线索初筛助手。请根据公司规则给询盘评级，并解释理由。不要因为采购量大就直接评为 A，信息完整度和产品匹配同样重要。

【结构化询盘】
{{structured_inquiry_json}}

【公司产品线】
{{product_lines}}

【评级规则】
A：产品明确、数量或项目规模明确、公司/国家可识别、与产品线匹配、需要 24 小时内业务跟进。
B：有真实需求迹象，但缺少规格/数量/交期/公司信息中的至少一项，需要补资料。
C：需求模糊、非目标产品、学生/个人零散咨询、明显群发或短期价值低。
Risk：疑似钓鱼、异常附件、敏感地区、要求异常付款、竞争对手、冒用知名公司域名。

请输出严格 JSON：
{
  "lead_grade": "A/B/C/Risk",
  "score": 0,
  "matched_product_line": "",
  "suggested_owner": "",
  "reasons": [],
  "risk_flags": [],
  "next_action": "",
  "questions_to_ask_customer": []
}

限制：不得建议报价；不得承诺交期、库存、付款条件；证据不足时降低等级并写明缺少哪些信息。
```

### 3. 首封回复草稿

```text
请为外贸业务员起草一封英文首封回复邮件。邮件必须保守、礼貌、可人工编辑，不包含价格、交期、库存承诺。

【客户询盘摘要】
{{structured_inquiry_json}}

【需要补充的问题】
{{questions_to_ask_customer}}

【公司公开产品说明】
{{public_product_blurb}}

要求：主题行不超过 12 个英文单词；正文 120–180 个英文单词；先复述客户需求，再提出 3–5 个补充问题；结尾说明 “our sales engineer will review the details”。

输出：
Subject: ...
Body:
...
```

### 4. 每日老板简报

```text
你是跨境贸易公司的销售运营助理。请根据今天的询盘分拣表生成中文日报，给老板和销售主管看。

【今日询盘数据 JSON】
{{daily_inquiry_rows_json}}

请输出 Markdown，包含：
1. 今日总览：总询盘数、A/B/C/Risk 数量、未分派数量。
2. 值得马上处理的 A 级线索：公司/国家/产品/理由/负责人。
3. 待补资料线索：缺什么、建议谁跟进。
4. 风险和异常：只写事实，不夸大。
5. 明天要处理的 3 件事。

限制：不输出客户隐私细节，例如完整电话、私人邮箱、附件内容；不输出报价、合同条款、成交概率百分比。
```

## 人类主管验收

- **字段准确率**：抽查产品、数量、国家、公司、邮箱、认证、目的港，首周目标 ≥80%。
- **评级一致率**：AI 等级与销售主管判断的匹配比例，首周目标 ≥70%。
- **响应速度**：A 级线索从进入邮箱到业务员看到提醒的时间。
- **漏判风险**：每天抽查 C 级和 Risk 级，确认没有高价值客户被压低。
- **系统纪律**：无自动外发、无自动报价、无自动改 CRM 阶段、无敏感附件入模。

## 不能自动化的事

- 客户承诺、价格、折扣、付款条件、佣金、税费、运费估算。
- 交期、库存、样品可发时间、生产排期。
- PI、PO、合同、变更条款、法务和出口管制判断。
- 客户内部图纸、合同、报价记录、历史成交价、供应链成本。
- 陌生压缩包、宏文件、可执行文件的自动下载或打开。
- 未经审核的对外邮件发送、CRM 金额/阶段/成交概率修改。

## 第一周试点方案

| 天数 | 工作内容 | 验收标准 |
|---|---|---|
| Day 1 | 定义字段、评级规则、产品线词表、负责人规则；准备 50 条脱敏历史样本 | 字段表和评级规则由销售主管确认 |
| Day 2 | 搭建邮件触发 → 文本抽取 → 结构化输出 → 表格写入 | 50 条样本进入分拣表，原文链接可追溯 |
| Day 3 | 加入去重、产品匹配、缺失信息识别 | 抽查 20 条，关键字段准确率 ≥80% |
| Day 4 | 加入 A/B/C/Risk 评级和首封回复草稿 | 销售主管抽查，评级一致率 ≥70% |
| Day 5 | 处理当天新增询盘，输出老板日报 | 业务员反馈节省时间；记录误判和规则修改清单 |

一周后再决定是否扩大到更多邮箱、表单和 CRM 入口。连续两周无重大误判后，才考虑增加自动提醒范围。
