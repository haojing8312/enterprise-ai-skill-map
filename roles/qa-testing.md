# 岗位｜QA/测试负责人

网页回归、测试执行、验收检查和质量门禁。

## 适用行业

- [软件/互联网/研发组织](../industries/software-internet.md)：2 个 Skill
- [内容/媒体/营销服务](../industries/content-media.md)：1 个 Skill
- [零售/电商/客服密集型](../industries/retail-ecommerce.md)：1 个 Skill

## 当前可用 Skill

|Skill|行业|场景|风险|一句话用法|
|---|---|---|---|---|
|[openai/codex](https://github.com/openai/codex)|软件/互联网/研发组织|r-and-d|medium|适合给会看代码的人提速：读仓库、修小 bug、跑命令都顺手，但前提是有人审、有人测。|
|[microsoft/playwright-mcp](https://github.com/microsoft/playwright-mcp)|软件/互联网/研发组织<br>零售/电商/客服密集型<br>内容/媒体/营销服务|r-and-d<br>marketing-content<br>operations-admin|medium|凡是需要“真的打开网页点一遍”的场景，它比纯文本 agent 靠谱。但它点的是真系统，所以账号隔离和审批不能省。|

## 岗位使用建议

- 把 Skill 当成工作流组件，不要当成完整岗位替代。
- 先写清输入、输出、审批人和失败回退方式。
- 能触达代码、账号、客户数据、合同或财务数据的能力，默认提高一个风险等级管理。
