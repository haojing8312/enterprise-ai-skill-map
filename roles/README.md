# Roles｜按岗位找 Skill

先问：企业里谁要用、谁负责验收、谁承担风险？

| 岗位组 | 当前 Skill | 说明 |
|---|---:|---|
| [老板/高管/业务负责人](executive-management.md) | 0 | 看方向、定边界、决定先在哪个流程试点。 |
| [策略/行业研究员](strategy-research.md) | 1 | 负责资料检索、竞品研究、行业分析和决策前材料。 |
| [开发工程师](engineering.md) | 4 | 读代码、改 bug、写测试、做小范围实现。 |
| [技术负责人/研发效能负责人](engineering-management.md) | 4 | 设计 AI 编码流程、验收标准、代码审查和发布边界。 |
| [研发平台/GitHub 管理员](engineering-platform.md) | 2 | 负责代码协作平台、Issue/PR 自动化、仓库权限和审计。 |
| [QA/测试负责人](qa-testing.md) | 2 | 网页回归、测试执行、验收检查和质量门禁。 |
| [AI 平台工程师](ai-platform.md) | 4 | 模型服务、网关、推理平台、MCP/Agent 工具接入。 |
| [基础设施/运维负责人](infra-ops.md) | 3 | 内网部署、GPU、边缘设备、系统稳定性和权限隔离。 |
| [成本治理负责人](cost-governance.md) | 2 | 模型调用、GPU 资源、网关计费和成本透明化。 |
| [AI 应用工程师](ai-application.md) | 1 | 把 Agent SDK、工具调用和业务流程做成可测试应用。 |
| [工作流架构师](workflow-architecture.md) | 3 | 设计多代理交接、状态流转、异常处理和可观察性。 |
| [数字员工架构师](digital-employee-architect.md) | 2 | 从组织能力出发，定义角色、工具、权限、交接和验收。 |
| [数字员工/Skill 设计师](digital-employee-designer.md) | 2 | 把可复用能力包装成可分发、可维护的 Skill 或模板。 |
| [内容/市场运营负责人](content-marketing.md) | 2 | 选题、写作、图文、渠道分发、数据复盘。 |
| [内容研究助理](content-research.md) | 1 | 为长文、短内容、课程和方案准备材料与出处。 |
| [知识库运营](knowledge-ops.md) | 3 | 整理文档、沉淀经验、维护可检索的知识资产。 |
| [文档处理专员](document-ops.md) | 1 | 格式转换、资料清洗、文档归档、引用准备。 |
| [法务/合规负责人](legal-compliance.md) | 1 | 合同、隐私、权限、数据使用和对外表述风险提示。 |
| [运营/行政/流程管理员](operations-admin.md) | 2 | 账号、权限、平台接入、制度、巡检和流程协调。 |
| [运营自动化负责人](operations-automation.md) | 1 | 把网页、后台、表单、报表等重复操作半自动化。 |
| [客服/客户成功负责人](customer-success.md) | 1 | FAQ、工单、用户记忆、实施手册和交付复盘。 |
| [产品经理](product-manager.md) | 1 | 定义需求、验收口径、用户记忆和体验边界。 |
| [解决方案/交付架构师](solution-delivery.md) | 0 | 把 Skill 组合成可交付的行业方案、课程或咨询包。 |
| [人力/培训运营](hr-training.md) | 0 | 招聘、培训、课程、员工手册和能力评估。 |
| [财务/税务/报销负责人](finance-tax.md) | 0 | 预算、报销、成本和税务材料准备；当前需要更多公开样本。 |
| [Skill 观察研究员](skill-research.md) | 1 | 发现、验证、记录外部 Skill 和社区工作流。 |
