# AI 独立站页面健康巡检员 / AI Website Health Auditor

> 每天替跨境老板盯住正在花钱、正在接客户的页面，先发现坏页面，再让人修。

## 适合谁

- 正在投 Google、Meta、TikTok、Pinterest 广告的独立站。
- Shopify / WooCommerce / DTC 品牌站。
- 靠官网拿询盘的外贸工厂、B2B 出口商、贸易公司。
- 页面多、活动多、SKU 多，但没有专人每天查页面健康的团队。

## 老板最常见的痛点

广告费每天在扣，客户点进来后，页面 404、打开太慢、按钮没反应、询盘表单提交失败、活动页还挂着过期优惠。投手看数据觉得转化差，业务觉得询盘少，老板最后才发现：页面没有接住人。

## 这个数字员工每天干什么

**输入：**
- 首页、产品页、分类页、广告落地页、活动页、联系页、博客页。
- 广告后台导出的投放 URL、sitemap、活动日历。
- PageSpeed / Lighthouse 报告、Search Console / GA4 聚合数据。

**输出：**
- 页面健康日报。
- P0/P1/P2/P3 问题清单。
- 截图、日志、状态码、跳转链、Lighthouse 报告等证据。
- 建议负责人：开发、运营、广告、SEO、设计。
- 修复后的复检结果。

## 可接手任务

1. 查广告页、产品页、联系页是否 404 / 5xx / 跳错页。
2. 查移动端速度、LCP、INP、CLS、首屏资源和第三方脚本。
3. 用 Playwright 测手机端按钮、表单、下载、Add to Cart / Contact Us / Get Quote。
4. 扫内部断链、外部失效链接、图片/CSS/JS 资源加载失败。
5. 查 title、meta、H1、canonical、noindex、sitemap、alt 等 SEO 基础项。
6. 对照活动日历找过期促销、旧展会信息、旧配送/退换货文案。
7. 做 axe-core 基础可访问性检查。
8. 每周复检并汇总高风险页面、已修复问题、重复问题。

## 能力模块

| 模块 | 老板听得懂的作用 | 公开工具 / Skill 线索 |
|---|---|---|
| URL 优先级 | 先查最值钱的页面：广告页、爆品页、联系页、询盘页 | Sitemap、Search Console API、GA4 Data API |
| 页面健康 | 看页面能不能打开、够不够快、有没有断链 | Lighthouse、PageSpeed Insights API、lychee、broken-link-checker |
| 移动端交互 | 手机上按钮能不能点、表单能不能发 | Playwright |
| 内容时效 | 活动页、Banner、优惠码、展会信息有没有过期 | 页面文本抽取 + AI 审核 |
| 分级日报 | 给老板摘要，给执行人 issue，修完有复检 | Markdown / Notion / GitHub Issues / 飞书多维表 |

## 工作机理

```text
重点 URL / 广告页 / sitemap / 活动日历 / PageSpeed / GSC 聚合数据
→ 状态码、断链、速度、按钮、表单、SEO、活动文案巡检
→ AI 去重、分级、写复现步骤、匹配建议负责人
→ 输出老板摘要 + issue 明细 + 截图/日志证据
→ 人类主管确认业务影响并派单
→ 修复后复检，记录已修复 / 未修复 / 无法复现
```

## 第一周试点

- **Day 1：** 选 20–50 个重点 URL，包含广告页、爆品页、联系页、询盘页、活动页。
- **Day 2：** 查 404、5xx、重定向链、断链、SEO 基础项。
- **Day 3：** 查移动端速度、首屏资源、第三方脚本。
- **Day 4：** 用 Playwright 测按钮、表单、下载、Add to Cart / Contact Us / Get Quote。
- **Day 5：** 人类主管复核前 10 个 P0/P1，修正规则。
- **Day 6：** 对已修复问题复检。
- **Day 7：** 固化页面健康日报模板，决定第二周是否扩大 URL 范围。

## 人类主管验收

- P0/P1 是否有 URL、设备、浏览器、时间、截图/日志、复现步骤。
- 严重等级是否合理：广告落地页优先于普通博客页。
- 是否误伤业务规则：noindex、跳转、下架可能是运营主动设置。
- 是否可派单：建议负责人、实际结果、期望结果、建议动作是否清楚。
- 修复后是否有复检记录。

## 不能自动化的事

- 改代码、改主题、改插件、改价格、改库存、改优惠码。
- 调整广告预算、投放策略、活动政策。
- 提交真实询盘、真实订单、真实支付，或使用真实客户身份测试。
- 读取/上传客户姓名、邮箱、电话、地址、订单、合同、报价。
- 绕过登录、验证码、地理限制、robots.txt 或服务条款。
- 输出法律、安全、隐私合规最终结论。
