# Start Here｜怎么快速找到合适的 Skill

如果你不知道从哪看，先按你的业务目标选入口。

## 1. 我是出海公司，要做 GEO / SEO / 内容增长

先看：[`tracks/overseas-geo-marketing.md`](tracks/overseas-geo-marketing.md)

推荐起步组合：

```text
RSSHub / Crawlee → n8n / Activepieces → Ghost → Plausible → Google Search Console
```

适合：B2B SaaS、独立站、工具型产品、AI 产品、跨境服务商。

## 2. 我要做营销自动化和线索获取

先看：[`tracks/marketing-automation.md`](tracks/marketing-automation.md)

推荐起步组合：

```text
Apollo / LinkedIn / 表单 → n8n / Activepieces → CRM / 邮件 → 报表复盘
```

注意：外联、群发、客户名单、报价和承诺信息必须有人审。

## 3. 我要搭内容生产与分发系统

先看：[`tracks/content-engine.md`](tracks/content-engine.md)

推荐起步组合：

```text
公开信号源 → 选题池 → 长文/短内容 → Ghost/公众号/社媒 → Plausible/平台数据
```

注意：AI 草稿不能直接作为对外发布内容。

## 4. 我要给研发团队上 AI 编码流程

先看：[`tracks/ai-coding-workflow.md`](tracks/ai-coding-workflow.md)

推荐起步组合：

```text
spec → AI 编码 → 测试 → code review → 发布检查
```

注意：不要把 AI 编码工具包装成无人值守开发员工。

## 5. 我只想找某个应用能不能自动化

看：[`app-automations/README.md`](app-automations/README.md)

这里按应用和业务场景列：Apollo、Google Search Console、Google Analytics、LinkedIn、X/Twitter、YouTube、Reddit、Gmail、Slack 等。

## 6. 我只想看单个 Skill

看：[`skills/`](skills/)

每个 Skill 都有人话卡片，包含适合谁、推荐用法、不适合什么、风险边界和来源。
